{% extends "base-module.html" %} {% block module %} # Git Git is a version control system written by Linus Torvalds in 2005 for the purpose of maintaining the Linux Kernel. It is available for all operating systems in some capacity, and is able to save the state of programming projects over the course of time, as well as merge the content of two programs. Git is a massive program, and has an extensive list of terminology which is important for understanding how to use it. This section will provide a glossary at the end, and a list of additional resources, for your convenience! ## Installing Git Select the tab with your current operating system, and follow the instructions to install git:
Linux
Mac
Windows
At the command line, run the appropriate command for your distribution [1]:

Debian/Ubuntu

$ sudo apt install git

Fedora

$ yum install git for Fedora < 21

$ dnf install git for Fedora >= 22

More distribution commands are available here.

Open the Terminal app, and run git --version. If you don't have git installed, it will prompt you to install it. You can install it according to the prompt, or install Homebrew and use the command provided underneath [1].

Homebrew

$ brew install homebrew

More options are available here.
